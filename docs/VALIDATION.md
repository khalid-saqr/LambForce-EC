# Validation strategy

## Software gate

Every pull request runs the repository audit, complete test suite, Ruff, bytecode compilation, wheel
build, and installed-wheel audit on Python 3.10, 3.11, and 3.12.

## Phase 1 hydrodynamic gate

The published-source workflow must demonstrate:

- exact source commit, notebook blob, and frozen input-registry binding;
- exact preservation of the six artery definitions and six signed harmonics;
- deterministic repeated generation of all twelve standardized payloads within one frozen
  environment;
- polynomial differentiation and classical isotropic Womersley checks;
- normalized linear-system residuals;
- radial quadrature, temporal, and spectral convergence at publication resolution;
- signed integration, exposure, WSS, isotropic, and anisotropy-increment consistency;
- historical-v2 regression against a cold execution of the exact notebook blob;
- a complete historical-versus-verified discrepancy report.

A software smoke profile cannot satisfy this scientific gate.

## Mechanics gate

Endothelial mechanics begin only after Phase 1 records are qualified and frozen. Production mechanics
then require mesh/time convergence, solver cross-verification, the registered parameter and
structural ensembles, and all nine decision gates in `protocol/decision_gates.yaml`.
